SJTU Agent

你是否觉得校园平台太多？

这是一个给交大学生用的校园助手。

它把 DDL 管理、信息检索和校园门户放进同一个入口。

了解功能 查看安装

为什么会需要它

  • 平台太多 Canvas、教学系统、教务处、my.sjtu.edu.cn 分散在不同入口。
  • DDL 太多 不同课程、不同平台的截止时间要自己盯、自己排优先级。
  • 消息太杂 通知、提醒、课表、实验安排混在一起，很难快速知道现在先做什么。

真实场景

这就是它在 Telegram 里每天真实出现的样子。

不是宣传词，也不是假数据。它会先告诉你哪些平台已经接好，再把当晚最重要的 DDL、课程、实验和建议一起发出来。

  • 先说明已经配置的平台和当前可用能力。
  • 晚上自动发学习日报，不用自己去各个平台翻信息。
  • 最后直接告诉你今晚先做什么，而不是只把消息堆给你。

一个入口，处理最常见的校园任务。

先管 DDL。再查信息。需要时继续进入校园门户。

A1

一站式 DDL 管理

汇总多个平台的作业和截止时间，排序、提醒、下载、读题可以连在一起做；如果最后 5 分钟还没交，会触发紧急保底提醒。

问答演示

用户：帮我看看这周哪些作业最急，再把大学物理题目下载下来。

Agent：这周最急的是 1 项今晚截止的作业，另外还有 2 项本周内截止。我已经把大学物理相关材料保存到 assignments/ 目录，如果你愿意，我可以继续直接读出第一题内容。

A2

信息检索

查课表、成绩、实验安排，也能搜教务处、水源和课程资料。

问答演示

用户：今天有什么课？顺便帮我查一下上学期 GPA。

Agent：你今天上午有一节课，下午没有课。上学期成绩我也已经查到了，课程名、成绩、绩点和学分会一起列出来，最后会给你该学期的加权 GPA。

A3

校园门户

需要进入 my.sjtu.edu.cn 办事时，可以继续往下走，而不是只停留在查询层面。

问答演示

用户：帮我去 my.sjtu.edu.cn 看看培养方案入口，继续找一下相关页面。

Agent：我会先进入 my.sjtu.edu.cn，再帮你定位到最相关的服务入口。如果中间需要继续点开具体页面，我会沿着门户流程继续往下走，而不是只停在首页。

A4

安装很简单

只需根据引导填写 jAccount 的账号密码，再设置大模型 API。

用户：我是第一次用，需要先做什么？

日报 + 紧急保底

Telegram

晚上自动汇总 DDL、课程、实验和通知；临近截止仍未提交时，会再发一条高优先级提醒。

作业 DDL 今日课程 下次实验 教务通知 今晚建议 DDL 紧急保底

边界

凭证保存在本地

账号信息写入本地文件，不默认上传到远端服务。

只做提醒，不代写代交

如果最后 5 分钟还没提交，只会发紧急提醒并尝试打开 Canvas 提交页，不会自动生成内容替你上交。

失败会讲清楚下一步

Cookie 过期或登录失效时，会直接提示下一步。

常见问题

我不会写代码，真的能用吗？

可以。这个项目的设计目标就是让你像聊天一样提需求，不要求你先懂命令行或接口。

它是只能查信息，还是能帮我做事？

两者都有。它既能查询，也能下载作业、读题、添加提醒、生成日报，以及在确认后执行作业提交等动作。

如果我只想用 Telegram，而不想开终端呢？

项目已经提供 Telegram Bot 接入，适合把日常对话和提醒都放到熟悉的聊天界面里。

为什么它比我自己开很多网页更省事？

因为它会替你决定去哪查、怎么查、结果怎么排优先级。你只保留“我要什么”，中间跳转都被折叠掉了。
